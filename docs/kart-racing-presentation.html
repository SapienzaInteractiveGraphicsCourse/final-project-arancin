Game Overview

Playable browser game, not only a static scene

The player selects a track, vehicle and mode, then drives in a real-time 3D scene with race state, HUD feedback and interactive controls.

  • Vehicles: Procedural Kart, Porsche Cayman GT4, Nissan Silvia S14 Kouki.
  • Tracks: Vegas Neon, Tropical Beach, Monaco Formula 1.
  • Modes: Race against AI or Time Trial.
  • Runtime systems: physics, track interaction, race manager, AI, camera, minimap, HUD and audio.

Runtime Frame Loop

Stable delta time and ordered updates

  • Targets 60 FPS with fixed delta for stable gameplay.
  • Caps large frame intervals to avoid simulation jumps.
  • Updates Tween.js runtime before systems that consume animated values.
  • Renders only after input, physics, race state, vehicle animation and UI are updated.
src/scene/preview/frameLoop.js
function animate(timestamp) {
  animationFrameId = requestAnimationFrame(animate);

  if (nextFrameTimestamp === 0) {
    nextFrameTimestamp = timestamp;
  }

  if (timestamp + FRAME_INTERVAL_EPSILON_MS < nextFrameTimestamp) {
    return;
  }

  const skippedIntervals = Math.max(
    0,
    Math.floor((timestamp - nextFrameTimestamp) / TARGET_FRAME_INTERVAL_MS)
  );
  nextFrameTimestamp += (skippedIntervals + 1) * TARGET_FRAME_INTERVAL_MS;
  frameRateMonitor.update(timestamp, TARGET_FRAME_RATE);

  const renderedIntervalMs = lastRenderedTimestamp > 0
    ? timestamp - lastRenderedTimestamp
    : TARGET_FRAME_INTERVAL_MS;

Scene Graph and Hierarchical Modeling

Core Three.js structure used throughout the project

The procedural kart is the main complex model. It is not a flat collection of meshes: it is a tree of groups where rotations and translations happen around meaningful local pivots.

Parent transforms

Moving the chassis group moves every child mesh: body, driver, wheels, headlights and engine.

Local pivots

Front steering pivots rotate only the front wheels; roll groups spin tire meshes.

Animated hierarchy

The driver head, shoulders, elbows, hands and steering wheel respond to driving state.

A (complex) hierarchical model

Driver Animation

Hierarchical animation that exploits the model structure

  • Driver has head pivot, shoulder pivots, elbow pivots and hand meshes.
  • Steering input changes head lean, shoulder effort, elbow angle and hand rotation.
  • Speed adds vibration and forward body lean.
  • This animation is entirely JavaScript-authored.
updateDriverAnimation()
if (this.driverHeadPivot) {
  this.driverHeadPivot.rotation.z = -steeringValue * speedRatio * 0.16;
  this.driverHeadPivot.rotation.y = steeringValue * (0.06 + speedRatio * 0.08);
  this.driverHeadPivot.rotation.x = -0.04 * speedRatio + vibration * 0.35;
}

this.driverArms?.forEach(({ side, shoulderPivot, elbowPivot, hand }) => {
  const handLift = Math.sin(steeringAngle + side * Math.PI * 0.5) * 0.035;
  shoulderPivot.rotation.x = -0.12 - speedRatio * 0.04 + handLift;
  shoulderPivot.rotation.y = side * (0.08 + effort * 0.1);
  shoulderPivot.rotation.z = side * 0.08 - steeringValue * 0.22;
  elbowPivot.rotation.x = 0.1 + speedRatio * 0.03 - handLift * 0.8;
  elbowPivot.rotation.z = -side * (0.04 + effort * 0.08);
  hand.rotation.x = steeringAngle * 0.55;
  hand.rotation.z = -side * 0.16 + steeringValue * 0.28;
});

Road Mesh Sampling

From curve samples to indexed BufferGeometry

  • For each progress value we sample center and tangent.
  • A right vector offsets left/right road borders.
  • UVs use cumulative distance for repeated asphalt texture.
  • Indices connect consecutive samples into triangles.
createRoadGeometry()
const center = curve.getPointAt(progress);
const tangent = curve.getTangentAt(progress).setY(0).normalize();
const right = getRightVector(tangent);

if (previousCenter) {
  cumulativeDistance += previousCenter.distanceTo(center);
}

const left = center.clone().addScaledVector(right, -roadWidth * 0.5);
const rightPoint = center.clone().addScaledVector(right, roadWidth * 0.5);
left.y = ROAD_Y;
rightPoint.y = ROAD_Y;

vertices.push(left.x, left.y, left.z, rightPoint.x, rightPoint.y, rightPoint.z);
uvs.push(0, cumulativeDistance / ROAD_UV_SCALE, 1, cumulativeDistance / ROAD_UV_SCALE);

Procedural Texture Set

Color, roughness and normal-style maps generated with CanvasTexture

  • Road texture uses seeded value noise and cracks.
  • Roughness map is derived from the generated height map.
  • Normal map samples neighboring heights to encode local slope.
  • Ground texture changes by track: sand, urban ground, dark Vegas base.
  • Textures use RepeatWrapping and anisotropy, connecting directly to the GPU texture/filtering lectures.
src/materials/proceduralTextures.js
const left = heightMap[y * TEXTURE_SIZE + wrap(x - 1, TEXTURE_SIZE)];
const right = heightMap[y * TEXTURE_SIZE + wrap(x + 1, TEXTURE_SIZE)];
const up = heightMap[wrap(y - 1, TEXTURE_SIZE) * TEXTURE_SIZE + x];
const down = heightMap[wrap(y + 1, TEXTURE_SIZE) * TEXTURE_SIZE + x];
const normal = new THREE.Vector3(
  (left - right) * NORMAL_STRENGTH,
  (up - down) * NORMAL_STRENGTH,
  1
).normalize();

Boost Pad Shader

Custom GLSL material animated by a JavaScript uniform

fragment shader idea
vec2 centeredUv = vUv - vec2(0.5);
float radius = length(centeredUv) * 2.0;
float angle = atan(centeredUv.y, centeredUv.x);
float pulseTime = uTime * 2.25 + uPhase;

float outwardWave = fract(radius * 1.75 - pulseTime * 0.42);
float ring = smoothstep(0.10, 0.0, abs(outwardWave - 0.5));
float corePulse = 0.55 + 0.45 * sin(pulseTime * 2.35);
float radialFade = smoothstep(1.0, 0.18, radius);
float spokeMask = smoothstep(0.62, 1.0, sin(angle * 6.0 + pulseTime * 1.6) * 0.5 + 0.5);
  • ShaderMaterial combines base color, accent color and opacity.
  • Rings and spokes are generated from UV radius and angle.
  • uTime is updated every frame from the track group.
  • Additive blending makes pads read as glowing gameplay objects.
modern shading languages, GPU textures, animation

Arcade Vehicle Physics

Custom controller instead of a rigid-body engine

  • Acceleration, braking, reverse and handbrake are implemented directly.
  • Exponential friction stabilizes speed decay.
  • Steering blends toward target values for smooth control.
  • Heading and position update in the XZ plane.
src/systems/ArcadeVehicleController.js
const reverseFactor = this.speed < 0 ? -1 : 1;
const handbrakeTurnMultiplier = inputState.handbrake ? 1.25 : 1;
const turnStrength =
  this.steering *
  this.performance.turnRate *
  surfaceGrip *
  this.speedRatio *
  handbrakeTurnMultiplier;
this.heading += turnStrength * reverseFactor * deltaTime;

const distance = this.speed * deltaTime;
this.position.x += Math.sin(this.heading) * distance;
this.position.z += Math.cos(this.heading) * distance;
Physics engine not required; custom arcade simulation fits the project

Conclusion

Why the project satisfies the Interactive Graphics assignment

Kart Racing Simulator combines a complete interactive 3D racing scene with the graphics concepts requested by the course.

  • It contains a complex team-authored hierarchical kart model.
  • It uses multiple real-time lights, shadows, imported and procedural textures, and a custom shader.
  • It provides meaningful user interaction through setup, driving, camera, lights, HUD and debug controls.
  • It implements animations in JavaScript, without importing animation clips.
  • It goes beyond a static scene by adding race state, AI, physics, collisions, minimap and audio feedback.
